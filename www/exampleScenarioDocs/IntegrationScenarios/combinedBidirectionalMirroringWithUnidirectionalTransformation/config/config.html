CCF-Example-Scenarios version 1 0 build 20

		This scenario combines the combined bidirectional mirroring
		scenario and unidirectional transformation scenario. What you
		can see from the wiring configuration is that the wiring looks
		exactly the same as in the bidirectional mirroring scenario. The
		only difference is the initial content in the synchronization
		information table that now also contains the data records to let
		the CCF synchronize the trackers defined in the unidirectional
		transformation scenario. Goal of this scenario is to show that
		in order to synchronize multiple repositories, no changes in the
		wirings have to be done. In fact, synchronization records can be
		added/deleted to the synchronization information table at
		runtime and the CCF will start/stop synchronization of the
		repositories specified in these rows.

This component is responsible for creating the identity mapping for newly created artifacts in the target system. It also updates the synchronization status for a particular repository. For SFEE systems the synchronization info is the last modified time of the last shipped artifact.
